Dreamer — Dream to Control

Paper: Dream to Control: Learning Behaviors by Latent Imagination — Hafner, Lillicrap, Ba & Norouzi, ICLR 2020, arXiv:1912.01603. This note is the Q&A from working through the method, so it leans on the parts that confused me: the p/q split, the actor–critic training, and how the value target relates to GAE.

📄 Paper Summary

Dreamer learns a world model (a compact "dream simulator") from past images, then trains the agent entirely inside the dream — it never needs the real environment to try out an action. The trick that makes this pay off: because the dream is a neural network, you can backpropagate gradients through imagined time to improve the policy directly, instead of using slow trial-and-error.

ComponentSymbolWhat it does
Representation model (p)p(s_t | s_{t-1}, a_{t-1}, o_t) Encode the real image o_t into a compact latent state s_t.
Transition model (q)q(s_t | s_{t-1}, a_{t-1}) Predict the next latent without seeing the image — this is what powers imagination.
Reward model (q)q(r_t | s_t) Predict the immediate reward from a latent state.
Action model (actor)q_φ(a_t | s_t) The policy: pick actions that lead to good futures.
Value model (critic)v_ψ(s_t) Predict the total future reward from a state.
flowchart LR subgraph A["(a) learn world model — REAL data"] o["images o_t"] --> P["p: encode → s_t"] P --> R["reward + image heads"] end subgraph B["(b) learn behavior — IMAGINATION"] s0["real s_t"] --> dream["dream H steps with q"] dream --> critic["train value v_ψ"] dream --> actor["train action q_φ"] end subgraph C["(c) act — REAL world"] img["o_t"] --> enc["encode → s_t"] --> pol["action model → a_t"] --> env["environment"] end A --> B --> C --> A

Headline result: on 20 visual control tasks (DeepMind Control Suite, from pixels), Dreamer beats prior model-based and model-free agents on data-efficiency, compute, and final score — using one set of hyper-parameters for all tasks.

🖊 My Key Notes — the Q&A that made it click

Q1. What do p and q actually mean?

Both are just distributions produced by neural nets. The letters mark a role, and the whole role is decided by one question: is the image available or not?

  • p = needs a real observation. The representation model looks at the actual image o_t to infer the state. Used when acting in the real world (perception with evidence).
  • q = does NOT need an observation. The transition model predicts the next state from only the previous state and action — no o_t inside it. Used when dreaming the future, where no image exists yet.
Real world:  o_5 (image) ─► p ─► s_5            (perception, evidence-based)
Dreaming:    s_5 ─(a_5)─► q ─► s_6 ─► q ─► s_7   (no images at all)

Training pushes q (image-free) to agree with p (image-informed) — that is the KL term in the world-model loss. Once they agree, the agent dreams forward without rendering a single pixel, which is why it can run thousands of imagined trajectories cheaply.

Kalman-filter analogy. Dreamer's skeleton is an old hidden-state model (Kalman filter / HMM): a hidden state you can't see, plus noisy observations that hint at it. Dreamer differs on three counts: the transition is a neural net (non-linear, real-valued state — a "non-linear Kalman filter"), it is conditioned on actions (so it reacts to decisions), and it predicts rewards (so it knows good from bad). Those last two are exactly what turn a passive observer into something you can plan with.
Q2. How are the value and action models trained?

It's actor–critic, entirely in imagination. Start from a real latent s_0 (from replay), dream H steps forward, then run two opposite updates.

Running example (horizon H=3, discount γ=1, dreamed rewards r_1=r_2=r_3=1, current critic guesses v(s_1)=10, v(s_2)=8, v(s_3)=6):

Value estimation — build the target V_λ (eqs 4–6)

Eq 4  V_R   = r_1+r_2+r_3 = 3            (sum rewards only — short-sighted, ignores the future)

Eq 5  k-step returns (sum k real rewards, then bootstrap with the critic):
      V_N^1 = r_1 + v(s_1)             = 1 + 10       = 11
      V_N^2 = r_1 + r_2 + v(s_2)       = 1 + 1 + 8    = 10
      V_N^3 = r_1 + r_2 + r_3 + v(s_3) = 1 + 1 + 1 + 6 = 9

Eq 6  V_λ = λ-weighted blend of ALL the k-step returns  (λ=0.5):
      V_λ = 0.5·11 + 0.25·10 + 0.25·9 = 10.25   ← the training target

Small k trusts the critic (low variance, biased); large k uses more real dreamed rewards (less bias, more variance). V_λ blends them — the classic TD(λ) trade-off.

Learning objective (eqs 7–8) — two opposite moves

  • Critic (eq 8): match the target. Regress v_ψ(s_0) toward 10.25, with the target held fixed ("stop gradient"). Not circular, because the target is built from real predicted rewards + the value at later states, so it is a bit more grounded than the raw guess.
  • Actor (eq 7): maximize the target. Raise the sum of V_λ along the rollout. The special Dreamer move is how: V_λ is a differentiable function of the policy weights φ (actions → states → rewards/values, every arrow a neural net), so it backprops the value straight through the imagined trajectory — an exact analytic gradient, no REINFORCE.

The requirement for that gradient is a differentiable action sample — the reparameterization trick (eq 3): a = tanh(μ_φ(s) + σ_φ(s)·ε), ε ~ Normal(0,1).

Action model ≠ action supervision. An "action model" is a thing that outputs actions; "action supervision" is a label for the correct action. Dreamer has the first, not the second. Its action model is trained by pushing actions up the value gradient (reward → value → gradient), so it learns good actions with no target action to copy. (Contrast: an imitation-learning VLA does have action labels and trains with ‖a − a*‖².)

Q3. Isn't eq 6 the same as GAE?

Same weighting machinery, different final object. Eq 5 is a single k-step return; eq 6 (V_λ, the λ-return) is the λ-weighted blend of all of them — that is the GAE-family object, not eq 5. The one difference from GAE is a baseline subtraction:

value  V_λ         = 10.25              (eq 6 — an absolute "how good is here")
GAE    advantage   = V_λ − v(s_0)
                   = 10.25 − 9 = 1.25   (relative — "how much better than expected")
Eq 6 V_λGAE
Typevalueadvantage = value − baseline
Needed forbackprop-through-dynamics (Dreamer)policy gradients / REINFORCE

Dreamer keeps the raw value because it backprops through the dynamics; it never needs the advantage's baseline-subtraction variance trick, which only exists to tame policy-gradient noise.

Q4. Where does eq 10 fit — is that the actor/critic training?

No. Dreamer has two separate objectives, run back-to-back each update step, linked by the latent states:

ObjectiveTrainsDataEqs
World modelencoder, transition, reward, image decoder (θ)real replay10
Behavioraction φ, value ψimagined rollouts7, 8

Eq 10 builds the simulator; eqs 7–8 train the agent inside it (world model frozen during behavior learning). Eq 10 has three terms — image reconstruction ln q(o_t|s_t), reward prediction ln q(r_t|s_t), and the KL term −β·KL(p‖q) that glues the image-free transition to the image-informed representation (this is what enables imagination).

Flight-simulator analogy. Eq 10 = build & improve the flight simulator (from real flights). Eqs 7–8 = train the pilot inside the simulator (from dreams). Acting = fly the real plane, record it, feed it back to improve the simulator. Loop forever.
Q5. Is the reward model the same as the critic? And what's its label?

Different networks. The reward model predicts the immediate reward at one step; the critic predicts the whole future sum. But they connect: the reward model's outputs are the ingredients that value estimation sums into the critic's target.

reward model → r_1,r_2,r_3  ──►  V_λ target (eq 6) = 10.25  ──►  trains critic (eq 8)

The reward model's label is the real reward from the environment, stored in the replay buffer when the agent acted — plain supervised regression, make q(r_t|s_t) match the true r_t. That's the one ground-truth signal everything is built on. The critic, by contrast, has no real teacher for "total future reward," so it bootstraps — its target V_λ is self-generated from rewards + its own guesses.

Q6. After training, is value estimation still used?

No — it's training-only scaffolding. At deployment, only the action model runs: encode image → latent → action. No imagination, no V_λ, and not even the value model.

deployment loop:   o_t ─► encode ─► s_t ─► action model ─► a_t ─► env
ComponentTrainingDeployment
Value estimation V_λ✅ target-builder❌ gone
Value model v_ψ (critic)✅❌
Action model q_φ (policy)✅✅ the only thing that runs
Encoder / transition / reward✅encoder only

Think of the critic as a coach during practice: it tells the actor which futures are good so the policy improves. On game day the coach stays off the field — the lessons already live in the policy's weights.

Takeaway for me: Dreamer = Ha & Schmidhuber's "train in the dream" idea, made fully differentiable. The world model (eq 10) gives you a compact, image-free latent simulator; the actor–critic (eqs 7–8) then learns by backpropagating value through imagined time rather than trial-and-error. The p/q split, the λ-return target, and the reward-vs-value distinction are the three things I kept re-deriving until they stuck.
