Tube Patch Masking — force the model to look across cameras

Sight-cone said which views may talk. Tube masking creates a reason they must.

The problem: even after sight-cone masking allows the right cross-view links, the model is lazy — it can often reconstruct a hidden patch from the same camera's other time frames (a temporal shortcut) and never bother using the other cameras. Tube masking removes that shortcut so the only way left is to look across views.

How tube masking works — the steps

  1. Decide to apply it. With probability p_tube this training sample gets a tube mask (otherwise it trains normally).
  2. Pick one view v*. Choose a single camera to punch a hole in.
  3. Pick a k×k window (k random in [l_min, l_max]) — but only from the co-visible set U: patches that at least one other view can see (using the sight-cone mask M_sc). This guarantees recovery is actually possible.
  4. Extend it through time into a "tube." Take that same window across every latent frame of v*. That spatio-temporal column is the tube.
  5. Replace the tube with pure noise in the video input x. With a nested probability p_tube^cond, blank the same tube in the conditioning y too, so the model can't peek at the answer there.
  6. Keep the true target and up-weight it. The model must still reproduce the real content of the tube, and its loss there is multiplied by λ_mask to push harder. With no within-view frames left to copy, recovery must route through the other N_v − 1 cameras.
Why the whole tube, and not just one frame? If you masked the window in only one frame, the model would happily copy it from the neighbouring frames of the same camera (they barely change frame-to-frame) — a temporal shortcut, and it would never learn to use the other views. Masking the window across all frames kills that shortcut, leaving cross-view attention as the only path. Try unchecking "whole tube" in the demo and watch the verdict flip.
Why sample only from the co-visible set U? If you blanked a region that no other camera can see, the information would exist nowhere — you'd be asking the model to hallucinate, which teaches nothing. Restricting the window to patches that another view does see (via the sight-cone mask) makes it a fair, solvable task: the answer really is available across views. This is exactly where the two masks connect.
